# Plan: Terminal Color Palettes

**Execute after:** plan-single-branch-dotfiles.md

## Current State

**iTerm profiles** (`iterm/`): Six Dynamic Profile JSON files. `0.base.json` ("Base") is the root
with all 16 ANSI colors, background, foreground, cursor, selection, bold, and UI color keys.
Other profiles inherit from Base (or Tomorrow Dark Mod) and override only what differs:
- Base — dark near-black background with custom ANSI colors
- NoBlur — inherits Base, tweaks blur/transparency only
- Tomorrow Dark Mod — inherits Base, replaces all colors with a Tomorrow Night variant
- Tomorrow Dark Mod Air — inherits Tomorrow Dark Mod, smaller fonts only
- Opaque — inherits Tomorrow Dark Mod, removes transparency
- Solarized Light Mod — inherits Base, replaces all colors with a light Solarized variant

**`bin/set-iterm-profile`** — already exists; sends `\033]50;SetProfile=$1\a` to switch profiles.

**`lib/002-colors.zsh`** — `LS_COLORS` is generated by `vivid generate jellybeans` on every shell
start; hardcoded to jellybeans regardless of active palette.

**`lib/010-aliases.zsh`** — `bat` alias hardcodes `--theme="Dracula"`.

**`home/config/starship.toml`** — uses named ANSI colors plus the literal `#444444` for separator
backgrounds (which looks wrong on light palettes).

## What "Alternate Colors" Means

A terminal color scheme has two layers:
1. The 8 ANSI accent colors (0–7) plus brights (8–15)
2. Semantic/UI colors — background, foreground, cursor, selection, bold — designed to look good
   against those accent colors

A "palette" is a complete pairing of both layers. The existing profiles already model this: Tomorrow
Dark Mod and Solarized Light Mod each replace all 16 ANSI colors plus all UI colors together.

## Recommended Approach

Continue using iTerm2 Dynamic Profile JSON as the storage format (already the established pattern).
Build a `palette` shell function that wraps `set-iterm-profile` and also updates `LS_COLORS` and
`BAT_THEME` to match.

## Implementation Steps

**1. Create `lib/003-palette.zsh`** — defines a `palette` function:

```zsh
typeset -A _palette_vivid _palette_bat
_palette_vivid=(
    "Tomorrow Dark Mod"   "jellybeans"
    "Solarized Light Mod" "solarized-light"
)
_palette_bat=(
    "Tomorrow Dark Mod"   "Dracula"
    "Solarized Light Mod" "Solarized (light)"
)

function palette {
    local name="${1:-Tomorrow Dark Mod}"
    set-iterm-profile "$name"
    export LS_COLORS="$(vivid generate ${_palette_vivid[$name]:-jellybeans})"
    export BAT_THEME="${_palette_bat[$name]:-Dracula}"
}
```

**2. Update `lib/010-aliases.zsh`** — change:

```zsh
# From:
alias bat='bat --theme="Dracula"'
# To:
alias bat='bat --theme="${BAT_THEME:-Dracula}"'
```

**3. Update `lib/002-colors.zsh`** — make vivid theme configurable:

```zsh
# From: export LS_COLORS="$(vivid generate jellybeans)"
# To:
export LS_COLORS="$(vivid generate ${VIVID_THEME:-jellybeans})"
```

And set `VIVID_THEME` from the palette system.

**4. Add new palette JSON files as needed** — e.g. a high-contrast dark variant. Each is a new
file in `iterm/` with `"Dynamic Profile Parent Name": "Base"` and a full color block.

**5. (Optional) Starship adaptation** — the hardcoded `#444444` separator background in
`starship.toml` looks wrong on light palettes. Use starship's `[palettes.*]` feature to define
swappable named colors, with the `palette` function writing a `~/.config/starship-palette.toml`
include. Low priority; acceptable to leave the separator off on light mode.

**6. Write `bin/iterm-colors`** — a utility to print a human-readable hex color table from a
Dynamic Profile JSON (the float-RGB format is hard to read). Useful for creating new palettes.

## Notes

- The `palette` function only affects the current terminal session. For a persistent default,
  set the default iTerm2 profile in iTerm2 preferences.
- `set-iterm-profile` only works in iTerm2 (uses a proprietary escape); on other terminals the
  function should degrade gracefully (skip that call, still set LS_COLORS/BAT_THEME).
- All changes go on the single branch (see plan-single-branch-dotfiles.md).
